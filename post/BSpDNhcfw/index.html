重读深入理解Java虚拟机（一）—— 读深入理解JVM之内存管理

2021-04-12 4 min read # JVM # 读书笔记

JVM内存管理

最近在读 深入理解 Java虚拟机，这里列出一下JVM内存管理部分大纲，还需配合该书进行学习。

运行时数据区

线程隔离

  • 程序计数器
    • Java方法
      • 虚拟机字节码指令的地址
    • Native方法
      • 计数值为空
    • 唯一一个没有规定OutOfMemoryError的区域
  • Java虚拟机栈
    • 描述方法执行的内存模型
      • 方法执行时创建栈帧
        • 局部变量表
          • 编译期可知的基本数据类型和引用类型
          • 64位长度的long和double类型占用2个局部变量空间（2 slot）
          • 编译期间大小分配完成，运行期间不会再动态更改
        • 操作数栈
        • 动态链接
        • 方法出口
      • 每个方法对应一个栈帧入栈和出栈的过程
    • 异常
      • StackOverflowError
        • 线程请求的栈深度超过虚拟机允许的
      • OutOfMemoryError
        • 虚拟机栈动态扩展时无法申请到足够的内存
  • 本地方法栈
    • 为Native方法服务
    • Sun HotSpot将本地方法栈和虚拟机栈合二为一

线程共享

  • 方法区
    • 用于存储已被虚拟机加载的类信息、常量、静态变量、即时编译器编译后的代码
    • Java虚拟机规范对方法区管理宽松
      • 不需要连续内存
      • 可以固定大小或扩展
      • 可以不选择垃圾收集
        • 对常量池的回收和对类型的卸载
        • 回收成绩难以令人满意
    • 运行时常量池
      • 存放class文件
        • Class文件有常量池信息，存放编译期生成的各种字面量和符号引用
          • 在类加载后进入方法区的运行时常量池
    • 发生区内存大小无法满足需要时
      • OutOfMemoryError
  • 堆
    • 存放对象实例
    • 垃圾收集的主要区域
    • 堆的划分
      • 内存分配的角度
        • 多个线程私有的分配缓冲区
      • 内存回收的角度
        • 根据分代收集法细分
          • 新生代
          • 老年代
    • 逻辑上连续即可
    • 可以固定大小也可以是可扩展的

直接内存

堆外内存

NIO类

  • 基于通道与缓冲区的I/O方式
  • 使用Native函数直接分配堆外内存
  • 通过Java堆中的DirectByteBuffer对象作为堆外内存的引用
  • 避免在Java堆和Native堆中来回复制数据

对象的创建

创建对象的过程

  • 遇到new指令，检查该类是否已经被加载、解析、初始化
  • 加载后确定了内存大小，从Java堆中划出内存
    • 内存分配方式
      • 指针碰撞
      • 空闲列表
    • Java堆是否规整选择分配方式
      • 根据垃圾收集器是否带有压缩决定
    • 并发情况下修改指针
      • 对分配内存空间的动作进行同步处理
        • 采用CAS配上失败重试的方式保证原子性
      • 将内存分配动作按照线程划分在不同的空间中
        • 每个线程在Java堆中预先分配一小块内存，本地线程分配缓冲
  • 将内存空间都初始化为零值
    • 成员变量默认的数据类型
  • 虚拟机对对象做设置，设置对象头
    • 对应类
    • 怎样找到类的元数据信息
    • 对象的哈希码
    • 对象的CG分代年龄

对象的内存布局

  • 对象头
    • 存储对象自身运行时数据（Mark Word）
    • 类型指针（指向类元数据的指针）
      • JVM通过类型指针确定对象是哪个类
      • 查找类时不需要经过对象本身
  • 实例数据
    • 对象真正存储的有效信息
    • 受到JVM分配参数策略和源码中字段定义顺序的影响
  • 对齐填充
    • HotSpot中对象的起始地址必须是8字节的整数倍
    • 通过对齐填充来补全

对象的访问定位

  • 通过栈上的reference数据来操作堆上的具体对象
  • 引用定位堆上对象的方式
    • 句柄访问
      • 堆的划分
        • 句柄池
          • 句柄对象
        • 实例池
          • 实例对象
      • 句柄对象
        • 到对象实例数据的指针
          • 指向实例池
        • 到对象类型数据的指针
          • 指向方法区
      • 优点
        • 稳定的句柄地址
          • 移动对象只要改变句柄对象的实例数据指针
    • 指针直接访问
      • 堆中实例数据中存在到对象类型数据的指针
        • 指向方法区的对象类型数据
      • 优点
        • 速度快
